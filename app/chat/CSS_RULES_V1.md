# 共通CSSルール v1（3アプリ横断）

## 1) 命名
- コンポーネント: `c-`（例: `c-button`, `c-card`）
- ユーティリティ: `u-`（例: `u-mt-2`, `u-text-muted`）
- 状態: `is-`（例: `is-active`, `is-hidden`）
- JSフック: `js-`（見た目スタイルを書かない）

## 2) デザイントークン（必須）
`:root` で以下を定義して使う（直書き禁止）
- 色: `--color-bg`, `--color-text`, `--color-border`, `--accent`
- 余白: `--space-1..6`
- 角丸: `--radius-sm/md/lg`
- 影: `--shadow-sm/md`
- 文字: `--font-sm/md/lg`, `--line-sm/md/lg`

## 3) 実装ルール
- 新規UIは必ず `c-*` で作る
- 既存修正は「触った箇所だけ」トークン化
- `!important` は原則禁止（例外はコメント必須）
- セレクタのネストは浅く（最大3段まで）
- 1コンポーネント1責務（ボタンでレイアウト調整しない）

## 4) レスポンシブ
- ブレークポイントは統一: `1024 / 768 / 480`
- モバイル優先で書く（`min-width` で拡張）

## 5) ダーク/ライト
- 色はトークン差し替えのみで対応
- コンポーネント側で固定色を持たない

- 対象アプリ:
- 変更ファイル:
- 共通ルール適用点:
- 非適用理由（あれば）:

---

## 適用方針（このリポジトリ）
- 以後のUI変更は本ルールを標準として適用する。
- 既存コードは一括置換せず、変更箇所から段階的に移行する。
- 互換優先のため、既存クラスは必要に応じて残しつつ `c-*` へ寄せる。
